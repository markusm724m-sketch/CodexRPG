**Graphics Testing & Quick Guide**

- **Purpose:** краткие шаги для проверки клиентских графических изменений (canvas, спрайты, звук, NPC).
- **Отключение Service Worker (если кэшируется старый скрипт):**

  1. Откройте DevTools (F12) → Console.
 2. Выполните:

     navigator.serviceWorker.getRegistrations().then(r=>r.forEach(s=>s.unregister()))
     location.reload(true)

- **Запуск сервера (локально):**

  В корне репозитория:

  ```bash
  python3 web/app.py
  ```

  Затем откройте в браузере http://localhost:5000

- **Проверка функционала:**

  - Создайте персонажа в UI (имя + класс) и нажмите «Create». Перейдите на экран игры.
  - Взаимодействие: клик по канвасу — игрок пойдёт в точку, появятся частицы и прозвучит короткий эффект.
  - Управление: WASD / стрелки — перемещение по тайлам.
  - NPC: в мире появятся NPC — они патрулируют случайно и двигаются по найденным путям.
  - Анимация: если браузер поддерживает SVG-спрайты, спрайты игрока и NPC будут показаны; иначе — fallback к простым фигурам.

- **Отладка:**

  - Если не видно изменений: перезагрузите страницу, проверьте, что `web/static/game.js` загружен (Network tab). Если кеширование мешает — очистите SW и кэш.
  - Для визуализации путей (debug): откройте `web/static/game.js` и временно включите отрисовку `n.path` (я могу добавить такую отладочную опцию по запросу).

- **Где смотреть код:**

  - Канвас и логика: `web/static/game.js`
  - SVG-спрайты: `web/static/sprites/player.svg`, `web/static/sprites/npc.svg`

Если хотите, могу автоматически включить отладочную отрисовку путей и уровней проходимости (вода/гора) — скажите, что предпочитаете.
